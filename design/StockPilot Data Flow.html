Section 01
Legend
Source · Milestone · Issue

Every row in this document tags an element with three things: where the data comes from, which milestone it lands in, and the canonical Linear/GitHub issue ID. Source chips use the same accent across the whole doc.

Data sources

yfinance Market data Historical OHLCV, last close, market index reads. Stack: yfinance via data/fetcher.py.
Anthropic AI engine Signal, confidence, reasoning text, daily portfolio recommendations. Stack: claude-sonnet-4 via analysis/ai_analyst.py.
Alpaca Paper trades Cash balance, position state, simulated execution. Stack: alpaca-py via trading/alpaca_client.py.
Local Derived / log Computed in-process (indicators, P&L math, runtime) or persisted locally (signals_log.json, watchlist).
UI only Presentation Pure dashboard chrome — no data dependency. Lives entirely in the Streamlit layer.

Milestones

M1 Data foundation May 27 – Jun 12 · yfinance fetcher, indicators, CLI entry point.
M2 Signal engine Jun 13 – Jul 3 · Anthropic prompt + parse, signal log.
M3 Paper trading Jul 4 – Jul 24 · Alpaca client, buy/sell on signal, portfolio persistence.
M4 Dashboard Jul 25 – Aug 14 · Streamlit UI, daily P&L, AI recommendations, discovery.
STO-NN Issue ID Linear ID, matches the GitHub issue (e.g. M1-02). 20 issues total across the 4 milestones.
